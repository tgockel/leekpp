Used checking certain expressions and potentially doing something about it.

By default, this function will throw an exception of type std::ex, but you can use it to do whatever you want by defining LEEK_ASSERT yourself.

Parameters
exprThe expression that was checked. This should be convertable to bool.
exThe exception type which would have been thrown if you would expect an assertion to throw an exception. The tokens here are limited to what you would find in the stdexcept header, but without the std:: prefix (for example, simply out_of_range and invalid_argument).
messageAn informative message which would be perfectly valid to pass to a function like printf (if you know the proper preprocessor incantations to do so). Examples of the contents of this variable are ("Invalid value %uz: must be between %zu and %zu", in_arg, lowest, highest), ("You look like you could use a %s", beer_name.c_str()), and ("The C preprocessor is %f%% a pain", 100.0). While this might seem really painful and insane to you, feel good knowing that probably everyone else (including the library author) agrees with you on that fact. Feel free to ignore this parameter if it pleases you.
