1 LeekPP: A Bloom Filter Library
2 ==============================
3 
4 A header-only library is a set of data structures and functions for dealing with
5  [Bloom filters](https://en.wikipedia.org/wiki/Bloom_filter).
6 It is hosted on [GitHub](https://github.com/tgockel/leekpp) and has online
7  [Doxygen documentation](http://tgockel.github.io/leekpp/) (which you might be looking at right now).

9 Usage
10 -----
11 
12  #include <leekpp/bloom_filter.hpp>
13 
14  #include <iostream>
15  #include <string>
16 
17  int main()
18  {
19  // Create an "ideally sized" Bloom filter with a 5% FPR and 1000 items
20  auto filter = leekpp::bloom_filter<std::string>::create_ideal(0.05, 1000);
21 
22  // Get items from the command line and insert them into the filter until we hit a blank
23  std::string ss;
24  while (std::cin >> ss && !ss.empty())
25  filter.insert(ss);
26 
27  // Test items from the command line until we EOF
28  while (std::cin >> ss)
29  std::cout << "Is '" << ss << "' in the filter? "
30  << (filter.count(ss) ? "maybe" : "no")
31  << std::endl;
32  }

41 ### Why is this called `leekpp`?
42 
43 The library is all about the Bloom filter, so I thought of the disgusting culinary masterpiece of the Bloomin' Onion.
44 Since Bloom filters have the property of false positives, one might consider them as a "leaky" data structure (a perfect
45  filter would not let anything unspecified through).
46 A leek is a type of onion _and_ is a homophone of "leak."
